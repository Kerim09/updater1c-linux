# Внешняя обработка проверки метаданных

Назначение: быстрый способ получить из базы:

```bsl
Метаданные.Имя
Метаданные.Синоним
Метаданные.Версия()
```

без полной выгрузки конфигурации через `DumpConfigToFiles`.

## Контракт с Python-приложением

Приложение запускает 1С примерно так:

```bash
1cv8 ENTERPRISE /F/path/to/base /Nuser /Ppassword /Execute MetadataProbe.epf /CU1C_META_OUT=/path/to/result.json
```

Обработка должна записать результат в один из файлов:

1. путь из параметра `/C`, если ты его распарсишь;
2. или fallback-файл `/tmp/u1c_metadata_probe_result.txt`.

Приложение понимает оба формата.

JSON:

```json
{"name":"Accounting", "synonym":"Бухгалтерия предприятия", "version":"3.0.195.40"}
```

Текст:

```text
name=Accounting
synonym=Бухгалтерия предприятия
version=3.0.195.40
```

После записи файла обработка должна закрыть 1С, иначе Python остановит процесс по таймауту.

## Как собрать EPF

1. Создай новую внешнюю обработку в Конфигураторе.
2. Добавь управляемую форму по умолчанию.
3. В модуль формы помести код из `ManagedFormModule.bsl`.
4. Сохрани как `MetadataProbe.epf`.
5. В приложении укажи путь к `.epf` на вкладке **Настройки программы**.
6. Метод определения поставь `auto` или `execute`.
